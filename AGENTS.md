# AGENTS.md — 选题猎手 (topic-finder)

## 这个项目是什么

一个帮创作者找小红书 / 抖音选题的工作流。把一个关键词 + 创作者自己的想法,加工成可以直接开做的「选题包」。

## 什么时候启动这个工作流

当用户表达出「找选题 / 看某个关键词火不火 / 我想讲 XX 有没有爆款角度 / 帮我判断这个选题能不能做」等意图时——哪怕没说"选题"二字,只要涉及小红书或抖音的内容方向、关键词热度、爆款角度,就按本工作流执行。

## 怎么执行

严格按 `./SKILL.md` 里的六步流程走。**开始前必读 `./profile.md`**(创作者档案,是所有判断的参照系)。不要跳过开场对话直接产出——第一步要先和用户摸清坐标和他手上的观点 / 素材。

## 依赖

本工作流需要外部数据工具拉小红书 / 抖音数据。**只有在真正取数时报"未安装 / 未配置"才停下让用户装,别拿自检命令去预判。**

本项目用的是 **`agent-reach`** 这个 skill(已加载,在 Codex 里经 `/agent-reach` 触发):

- **取数走 `xhs` 命令**:支持搜索、读笔记正文、读评论(含评论正文采样),四个指标(点赞 / 收藏 / 评论 / 转发)能分开拿。第二、四步都用它。
- **不要用 `agent-reach doctor` 之类的自检命令判断 skill 在不在**——该命令不存在,失败 ≠ skill 没装(skill 已加载)。要看状态用 `xhs status`。
- `xhs` 会**访问小红书,需要网络权限**,且依赖有效的登录 / cookie。遇到取数失败,先查网络和 cookie 这两项,而不是断定工具缺失。
- (抖音若要取数,需另配对应命令 / 工具;`xhs` 仅限小红书。)

方法论侧的依赖说明见 `./SKILL.md` 的「开始前:检查依赖」。
