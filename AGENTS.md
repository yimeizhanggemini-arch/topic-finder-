# AGENTS.md — 选题猎手 (topic-finder)

## 这个项目是什么

一个帮创作者找小红书 / 抖音选题的工作流。把一个关键词 + 创作者自己的想法,加工成可以直接开做的「选题包」。

## 什么时候启动这个工作流

当用户表达出「找选题 / 看某个关键词火不火 / 我想讲 XX 有没有爆款角度 / 帮我判断这个选题能不能做」等意图时——哪怕没说"选题"二字,只要涉及小红书或抖音的内容方向、关键词热度、爆款角度,就按本工作流执行。

## 怎么执行

严格按 `./SKILL.md` 里的六步流程走。**开始前必读 `./profile.md`**(创作者档案,是所有判断的参照系)。不要跳过开场对话直接产出——第一步要先和用户摸清坐标和他手上的观点 / 素材。

## 依赖

本工作流需要一个能拉小红书 / 抖音数据的外部工具。**不要假设它已经装好**——换台机器、换个人用,它就不在了;也别拿自检命令去预判它在不在。走到第二 / 四步真正需要数据时再调用,只有这个**真实的取数调用**明确报"未安装 / 未配置",才停下来处理依赖(见下面的降级规则)。

推荐工具是 **Agent Reach**(开源免费,支持 Codex):

- 安装:`pip install agent-reach`,再运行 `agent-reach install`(仓库与详细说明见 https://github.com/Panniantong/Agent-Reach)。
- **取数命令不要在本文件里硬编码。** Agent Reach 自带 SKILL.md,装好后 agent 读它就知道小红书当前走哪条路(目前桌面走 OpenCLI、服务器走 xiaohongshu-mcp,旧的 `xhs` / xhs-cli 后端已停更)。本文件只描述需求:"需要小红书的搜索 / 读笔记 / 读评论,四个指标(点赞 / 收藏 / 评论 / 转发)尽量分开拿",具体命令交给工具——这样它换后端,你这边不用改。
- **看状态用 `agent-reach doctor`**:它会列出每个渠道是否就绪、当前走哪个后端。
- 小红书取数需要**有效的登录态 + cookie**,可能还需要网络权限。导 cookie 建议用小号、别用主账号。取数失败先查 cookie 和网络这两项,而不是直接断定工具缺失。

**降级规则(关键,别让流程死在这里):**

- **完全取不到数据**(工具没装 / 没配 / 取数失败):不要直接停,转入 `./SKILL.md` 「开始前:检查依赖」里写的「手动模式」——让用户自己去平台搜,把前 5–8 条爆款的标题 + 赞 / 藏 / 评 / 转、再加几条高赞评论贴过来,照常往下跑。
- **能取热度但取不到评论正文**:第四步的评论挖掘跳过,用饱和度 / 观点核查顶上,并告诉用户少了这块。
- **抖音**:目前没有稳定的自动取数后端,直接走手动模式(Agent Reach 当前主要解决小红书)。

方法论侧的依赖说明见 `./SKILL.md` 的「开始前:检查依赖」。
