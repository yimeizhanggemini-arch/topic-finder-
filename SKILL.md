# 选题猎手 (Topic Finder)

把一个粗糙的想法或一个关键词,加工成"可以直接开做"的选题包。

这个 skill **不负责凭空发现热点**(那是热度数据工具的活,见「调用热度工具」)。它负责的是更难、也更值钱的一步:把一个通用方向 + 创作者自己的想法,翻译成"适合这个账号、这个平台、这个目标,而且别人还没做透"的选题。

## 开始前:先读档案

每次运行前,先读取同目录下的 `profile.md`。它定义了当前账号的平台、赛道、目标、人设和历史。**所有判断都要以档案为参照系**——同一个关键词,对 AI 科普号和情感号是完全不同的选题。

如果 `profile.md` 不存在或带 ⚠️ 的关键字段为空,先提示用户补全,再继续。

## 开始前:检查依赖

这个 skill 靠外部数据工具才跑得最顺(返回小红书 / 抖音的热度数据 + 评论正文)。**具体用哪个工具、在本项目里怎么调用,见 `AGENTS.md` 的「依赖」。** 但要记住:工具是"加速器"不是"开关"——拿不到数据时,这个 skill 仍然能用「手动模式」跑完(见下),因为它真正值钱的是方法论,不是取数本身。

判断工具可不可用,**别用"探针命令"去预判,容易误报**:

- **不要**用"先猜工具在不在"来决定走不走:真正的判断只有一个——走到第二 / 四步真要数据时,直接按 `AGENTS.md` 的方式发起**真实的取数调用**,看它成不成。(想主动看状态,用工具自带的状态命令,如 Agent Reach 的 `agent-reach doctor`;不要随手编一个子命令、它一失败就断定工具没装。)
- **取数调用明确报"未安装 / 未配置"时,不要停下干等,转入「手动模式」**:告诉用户暂时没有自动取数工具,请他去对应平台(小红书 / 抖音)搜一下这个关键词,把前 5–8 条爆款的标题 + 赞 / 藏 / 评 / 转、再加几条高赞评论贴过来;拿到后照常走第二 / 四步。绝不假装能取到数据,但也别让整个流程死在这里。
- 如果工具能取热度但取不到**评论正文**(有些工具只给点赞 / 收藏 / 评论"数量",给不了正文),第四步的评论挖掘就跳过,用饱和度 / 观点核查顶上,并告诉用户少了这块。

## 流程总览

```text
关键词进来
 │
 ▼
1 · 开场对话     定坐标(平台/目标/赛道)+ 摸需求(只有词?还是也有观点/经历?)
 │               └ 用户给的观点/经历,决定产出准不准、是不是只有 TA 能做
 ▼
2 · 快速摸底     拉热度 + 列 5–8 条相关爆款                       〔Agent Reach〕
 │
 ▼
3 · 定方向       干货 or 情绪?  ——  明显就走,骑墙就问
 │
 ├──▶ 4a 干货道   查饱和度(数量/时间/角度雷同?)+ 扒评论里的"提问"  〔Agent Reach〕
 │
 ├──▶ 4b 情绪道   查观点是否被说烂 + 扒评论里的"分歧"               〔Agent Reach〕
 │                (两条道汇合)
 ▼
5 · 按目标打分   涨粉 → 破圈传播力优先(而非专业深度)
 │
 ▼
6 · 输出选题包   推荐理由 + 差异化角度(套角度模板)+ 标题钩子
                 情绪文结尾附「四选一」确认清单
```

## 整体流程

关键词 / 想法进来后,依次走六步。**核心原则:宁可问清楚,也不要猜。** 但"问"要花在刀刃上——

- **开场(第一步)做一次充分交互**:把坐标和用户自己的想法一次性摸清。
- 之后 Agent 自己往下跑,**只在两个真正该停的地方再问用户**:① 第三步遇到骑墙词;② 走情绪线但开场没拿到观点。
- 其余步骤(摸底、打分、出包)是 Agent 该独立完成的,不要逐步打断用户。

---

### 第一步:开场对话(定坐标 + 摸需求)

不要拿到一个关键词就闷头跑。**一个光秃秃的关键词信息太薄**——基于它硬产出一堆内容,要么泛泛而谈,要么不准。开场先和用户聊清两件事:

**A. 定坐标**(多数从 `profile.md` 读,缺了才问):

- 平台:小红书 还是 抖音?(逻辑不同,见「平台差异」)
- 目标:涨粉 还是 变现?(决定第五步打分权重)
- 赛道:从档案读;关键词明显跨赛道才跟用户确认。

**B. 摸需求(关键)**:用户手上到底有什么?给一个好选的菜单——

> 关于「关键词」,你手上是哪种情况?
> ① 只有这个关键词,你帮我从零判断、挖角度
> ② 关键词 + 我已经有自己的观点 / 态度(我来补充)
> ③ 关键词 + 我有亲身经历 / 具体案例(我来补充)
> ④ 以上都有,我一起给你

- 选 ②③④ 的:请用户把观点 / 经历贴出来。**这些是整份产出"准不准、是不是只有 TA 能做"的关键**——尤其情绪文,用户的观点就是灵魂;而第六步确认清单里的"真实体验""独特视角",本来就只能来自用户,长不出来。把这些记下来,后面每一步都带着它判断。
- 选 ① 的:照常往下走,但第四步会更依赖挖掘,产出也会更"通用"一些;走到情绪线时再回头补问观点。

---

### 第二步:快速摸底

在分干货 / 情绪之前,先给用户一个"这方向值不值得碰"的直观判断。调用热度工具(见「调用热度工具」),产出两样:

1. **一句话结论**:这个方向现在热不热、是在涨还是在退。
2. **5–8 条相关爆款**:把这个关键词下近期跑得好的标题列出来。

这一步的产出**直接喂给第三步**:看着这 8 条爆款,大多是"高收藏的教程"还是"高评论的观点",方向就有了眉目。所以摸底不是孤立开场,它是定方向的原料。

---

### 第三步:定方向 —— 干货 or 情绪(带置信度)

选题分两类,加工方式完全不同:

- **干货文**:靠"有用"取胜。命门是**稀缺**——市面上找不到,或讲得更透。
- **情绪文**:靠"有共鸣 / 有态度"取胜。命门是**独特**——一个别人没说过、或不敢说的观点。

判断依据:综合关键词信号 + 第二步那批爆款的形态。

- **明显偏干货**:关键词含"教程 / 方法 / 怎么 / 工具 / 清单 / 步骤 / 避坑 / 测评",或爆款普遍高收藏。
- **明显偏情绪**:关键词含第一人称("我")、价值判断("最 / 千万别 / 后悔")、情绪词("焦虑 / 破防 / 治愈"),或爆款普遍高评论。
- **骑墙**:中性词,既能做客观盘点也能做主观表态(例:"AI 会不会取代人")。

**高置信度直接走对应分支。骑墙词,不要猜,提问用户**:

> 「XX」这个方向,你想做成 ① 教人实操的干货,还是 ② 表达观点 / 引发共鸣的情绪文?两条路的检查标准不一样,先定方向。

---

### 第四步:分流深挖

方向定了,走不同路径。两条路都再次调用热度工具,而且**评论区是这一步最值钱的料**——它是真实需求的原始现场,没人帮你提炼过,竞争对手也懒得扒。(前提是数据工具能取到评论正文,见「开始前:检查依赖」;取不到就跳过下面的评论部分,用饱和度 / 观点核查顶上。)

#### 4a 干货道 → 查"是不是烂大街了" + 扒评论里的"提问"

**查饱和度**(用可观察信号,别凭感觉):

- 头部内容有多少篇?(供给量)
- 大多什么时候发的?——都是半年以上的老内容 = 有翻新空间,不算真饱和。
- 角度是否高度雷同?——雷同 = 还有差异化空隙(机会);角度已百花齐放才是真红海。
- 最近还有没有新内容在涨?——有 = 需求还在;没有 = 伪需求或已退潮。

**扒评论提问**:让热度工具拉这个方向下爆款的高赞评论,专挑里面的提问。取数提示词:

> 拉「关键词」下几条爆款笔记的高赞评论,挑出其中读者的提问 / 困惑 / 追问(例如"那 XX 情况怎么办""新手适用吗""有没有免费替代")。这些没被原作者解答的问题,就是现成的子选题。

产出:饱和度结论 + 还剩哪个差异化空隙 + 一批从评论里挖到的真实子选题。

#### 4b 情绪道 → 查"观点是否被说烂" + 扒评论里的"分歧"

**先确认观点**:

- 第一步已拿到观点(用户选了 ②③④)→ 直接用它去查。
- 第一步没拿到(用户选了 ①)→ 现在补问:

> 情绪文的灵魂是观点。关于「XX」,你的态度 / 观点具体是什么?(越具体、越反直觉越好)

**查观点新鲜度**:让热度工具查这个角度现在的主流声音、有没有反方、是否已被讨论到疲劳。

**扒评论分歧**:取数提示词:

> 拉「关键词」下几条爆款笔记的评论区,找出争议点 / 分歧:大家在为什么吵?哪条"反对"评论被顶得很高?哪个"我也是"被疯狂附和?争点本身就是下一篇情绪文的钩子。

产出:观点是否还新鲜 / 有没有更尖锐的切入 + 一批从分歧里挖到的钩子。

---

### 第五步:按目标打分(过滤)

到这里有一批"能做"的选题。但"能做" ≠ "该做"。按 `profile.md` 的目标排序:

- **目标是涨粉**:涨粉靠**传播**。把"破圈传播力"高的排前面,而非"专业深度"高的。同样是 AI 科普,"GPT 新功能详解"(精准但圈层窄)应排在"我用 AI 把工资翻倍,普通人也能学"(有破圈钩子)之后。判断标准:这个选题能不能让人想评论、想转发、想 @ 朋友?
- **目标是变现**:变现靠**精准**。把"和变现强相关、受众精准"的排前面,哪怕圈层窄。

**不要给所有目标推一样的选题**——那样这个 skill 就没意义了。

---

### 第六步:输出选题包 + 收尾确认

不要只丢一个标题。每个推荐给一个"包":

- **推荐理由**:背后的数据 / 逻辑(第四步的核查结论)。
- **差异化角度**:具体切入点在哪——整个包最值钱的部分。**不靠灵感,套下面的「角度模板」逐一问一遍**:
  - **换对象**:同一个干货,换一个人群重做(「XX,但专门写给完全不会的人 / 妈妈辈 / 转行的人」)。
  - **换场景**:把抽象方法塞进具体情境(「我用这个方法,3 天搞定了 XX」)。
  - **反着说**:主流都教"怎么做",你来说"别这么做 / 我后悔做了 XX"——反共识天然有传播力。
  - **抬 / 降门槛**:要么"进阶版,90% 的人不知道的 XX",要么"零基础也能上手的 XX"。
  - **结果前置**:把最炸的真实结果 / 数据放标题(「我靠 XX 涨了 1 万粉」,前提是真实)。
  > 对 AI 科普 + 涨粉,「换对象(写给小白)」和「反着说」是破圈最稳的两个。
- **标题钩子**:3 个备选。小红书重标题本身,抖音重前 3 秒口播。
- **封面 / 开头思路**:一句话即可。
- **预期 & 风险**:可能的数据表现,以及风险点。

**收尾确认清单**:创作者自己拍板,skill 不替他下结论,只逼他过一遍这道关——人忙起来最容易跳过的就是这步。两类文走不同的关:

**情绪文** → 结尾固定附上这四条,满足任意一条即可做,一条都打不上勾就别做:

> ⚠️ 最后由你确认(我不替你判断):这个选题里,真正"属于你"的是哪一个?
> ☐ 真实的体验——你亲历的,别人编不出来
> ☐ 独特的视角——同一件事,你的角度和主流不同
> ☐ 强烈的情绪——能让人愤怒 / 感动 / 爽 / 扎心
> ☐ 个人的共鸣——读者会说"这写的就是我"

**干货文** → 它对应的关是"稀缺性",第四步 4a 已经查过:能不能讲出别处难找、或你讲得更透的东西?过了就做。(真实体验 / 独特视角 若也占上,是加分。)

## 调用热度工具

第二、四步需要平台数据。调用你配置的热度数据工具(如 **Agent Reach**,或任何能返回小红书 / 抖音数据的工具)。

**关键:不要问"热度高不高"。热度不是一个数,是一组信号。** 让工具按四个维度分别返回,因为它们各说明一件不同的事:

- **点赞**:基本认可,门槛最低,最不值钱(可能只是被推得多)。
- **收藏**:"有用,留着以后看"——**干货文的命门**。收藏 / 点赞比越高,干货含金量越高。
- **评论**:"我有话说"——**情绪文的命门**。能引发争论 / 共鸣的,评论一定高。评论区本身也是下一批选题的金矿(见第四步)。
- **转发**:"我要让别人看到"——传播力最强、最稀有的信号,**涨粉的核心指标**。

推荐的取数提示词模板:

> 分析关键词「XX」在 [平台] 近 30 天的表现。我要的不是单一热度,而是:这个词下的爆款,点赞 / 收藏 / 评论 / 转发大概什么量级和比例?是收藏占比高,还是评论占比高?这个词整体在涨还是在退?据此判断它更适合干货还是情绪方向。

注:具体能取到哪些维度、准不准,取决于你接的数据源。即使只能给定性结论("这个词下收藏普遍很高")也够用——你要的是方向,不是论文精度。

**没有数据工具时(手动模式):** 把上面这套要的东西,变成让用户去平台手动取——请他搜这个关键词,把前 5–8 条爆款的标题、各自的赞 / 藏 / 评 / 转、再加几条高赞评论复制贴回来。你在这批人工贴来的数据上做同样的判断(收藏占比高还是评论占比高、在涨还是在退、评论里有哪些提问 / 分歧)。结论会粗一些,但方向依然立得住,整个流程不会卡死。

## 平台差异

同一个选题,两个平台的包装不同。锁定平台后,套对应标准:

**小红书**
- 核心逻辑:搜索 + 种草。很多流量来自用户主动搜索。
- 命门:**标题**(决定点不点进来)+ **收藏**(决定算不算好内容)。
- 选题偏好:能被搜索的、有保存价值的、标题能戳中具体场景的。

**抖音**
- 核心逻辑:推荐 + 完播。系统会推给可能不认识你的人。
- 命门:**前 3 秒钩子**(决定划不划走)+ **完播率 + 转发**。
- 选题偏好:开头能瞬间抓人的、有强冲突或强利益点的、适合短平快表达的。

## 一句话原则

这个 skill 的价值不在"找到热点",而在把热点 + 创作者自己的想法,翻译成"只有这个账号能做、且能达成目标"的选题。**开场把人的观点摸清,中途信息不够就问,其余自己跑**——这是它比通用热榜高级的根本原因。
